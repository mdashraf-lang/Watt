## نظرة عامة على أنواع المستخدمين

```
┌─────────────────────────────────────────────────────────────────┐
│                    مستخدمو تطبيق واط                           │
├───────────────┬───────────────┬───────────────┬─────────────────┤
│    الزائر     │    العميل     │   المستثمر    │    الإدارة      │
│               │               │  (المضيف)     │                 │
│ تصفح الخريطة │ الحجز والشحن  │ مشاركة الشاحن │ قبول الطلبات   │
│ عرض المحطات  │ إدارة المحفظة │ كسب الدخل     │ التحقق بالأجهزة│
│ بدون حساب    │ حساب كامل    │ حساب كامل     │ إدارة الشبكة   │
│               │               │ + تبويبات خاصة│                 │
└───────────────┴───────────────┴───────────────┴─────────────────┘
```

| نوع المستخدم | هل يحتاج حساباً؟ | القدرة الرئيسية | عدد التبويبات |
|-------------|-----------------|----------------|---------------|
| الزائر | لا | تصفح الخريطة والمحطات | ٤ (الحجوزات والمحفظة مقفلتان) |
| العميل | نعم | حجز المحطات، إدارة المحفظة | ٤ |
| المستثمر | نعم (معتمد) | كل ميزات العميل + إدارة الشاحن الخاص | ٥ |
| الإدارة | نعم (معيَّن) | قبول المستثمرين، إدارة المستخدمين والشبكة | ٤ |

## التدفق ١ — الزائر (Guest)

الزائر هو كل من يفتح التطبيق دون تسجيل الدخول. يمكنه التصفح بحرية لكن لا يستطيع الحجز أو استخدام المحفظة.

### رحلة الزائر الكاملة

```
فتح التطبيق
    │
    ▼
شاشة البداية (٣ شرائح تعريفية)
    │  التمرير أو الضغط على "تخطي"
    ▼
شاشة تسجيل الدخول
    │  الضغط على "تصفح كضيف"
    ▼
┌──────────────────────────────────────┐
│      الرئيسية للزائر (تبويب الخريطة) │
│  • عرض جميع المحطات العامة           │
│  • مرمّزة بالألوان حسب الحالة:       │
│    🟢 متاح    🟠 مشغول               │
│    🔴 عطل    ⚫ غير متصل             │
│  • قائمة بالمحطات القريبة           │
│  • بحث المحطات بالاسم               │
└─────────────────┬────────────────────┘
                  │ النقر على دبوس أو عنصر قائمة
                  ▼
    شاشة تفاصيل المحطة
    • الاسم، العنوان، التقييم
    • السعر لكل kWh
    • أنواع الموصلات وحالتها
    • ساعات التشغيل والمرافق
    • زر "الاتجاهات" (يفتح خرائط)
    • زر "احجز الآن" ← يحوّل لتسجيل الدخول
                  │
                  │ النقر على تبويب الحجوزات أو المحفظة
                  ▼
    شاشة الزائر المقفل
    • تعرض ما يفتقده
    • زرّا "تسجيل الدخول" و"إنشاء حساب"
                  │
                  │ النقر على تبويب الملف الشخصي
                  ▼
    شاشة ملف الزائر
    • دعوة "إنشاء حساب"
    • معاينة المزايا
```

### ما يستطيع الزائر فعله

| الإجراء | متاح؟ |
|---------|--------|
| عرض الخريطة والمحطات | ✅ نعم |
| قراءة تفاصيل المحطة | ✅ نعم |
| الحصول على الاتجاهات | ✅ نعم |
| البحث عن محطات | ✅ نعم |
| حجز خانة شحن | ❌ لا — يتطلب تسجيل الدخول |
| عرض المحفظة | ❌ لا — يتطلب تسجيل الدخول |
| عرض سجل الحجوزات | ❌ لا — يتطلب تسجيل الدخول |

### مسار الترقية: زائر ← عميل

من أي شاشة مقفلة أو ملف الزائر، اضغط **تسجيل الدخول** أو **إنشاء حساب** للانتقال لتدفق التسجيل.

## التدفق ٢ — العميل

### ٢.١ التسجيل

```
شاشة تسجيل الدخول
    │
    ├─── لديك حساب بالفعل؟
    │         │
    │         ▼
    │    أدخل البريد الإلكتروني + كلمة المرور
    │         │
    │         ▼
    │    التطبيق يحمّل ملفك الشخصي
    │         │
    │         ▼
    │    الرئيسية (تبويب الخريطة) ✅
    │
    └─── مستخدم جديد؟
              │
              ▼
         شاشة إنشاء الحساب
              │
         أدخل:
         • الاسم الكامل
         • البريد الإلكتروني
         • كلمة المرور (٦ أحرف على الأقل)
              │
              ▼
         اضغط "إنشاء الحساب"
              │
         يُنشأ ملفك الشخصي تلقائياً
              │
              ▼
         الرئيسية (تبويب الخريطة) ✅

    ─── أو استخدم تسجيل اجتماعي ──────────────
    │
    ├─ "المتابعة بـ Google"
    │       ← متصفح داخلي يفتح OAuth من Google
    │       ← الموافقة ← المتصفح يغلق ← الملف يُنشأ
    │       ← الرئيسية ✅
    │
    └─ "تسجيل الدخول بـ Apple" (iOS فقط)
            ← نافذة Apple الأصلية
            ← الموافقة ← الملف يُنشأ
            ← الرئيسية ✅
```

### ٢.٤ إدارة الحساب

#### المحفظة — شحن الرصيد

```
تبويب المحفظة
    │
    ▼
┌─────────────────────────────┐
│  💰 5.000 ر.ع               │
│  رصيد المحفظة               │
│                             │
│  الجلسات: 12                │
│  إجمالي kWh: 144            │
│  المنفق: 4.032 ر.ع           │
│                             │
│  [⬆️ شحن]                   │
└─────────────────────────────┘
    │ اضغط "شحن"
    ▼
نافذة الشحن
    اختر المبلغ:
    [1 ر.ع] [2 ر.ع] [5 ر.ع]
    [10 ر.ع] [20 ر.ع] [50 ر.ع]
    │
    ▼
شاشة التأكيد:
    الرصيد الحالي:  5.000 ر.ع
    المبلغ المضاف: +10.000 ر.ع
    الرصيد الجديد: 15.000 ر.ع
    الدفع عبر:      ثواني
    │
    اضغط "تأكيد الشحن"
    ▼
نافذة دفع ثواني تفتح
    │ إتمام الدفع
    ▼
✅ "تم الشحن بنجاح — أضفت 10.000 ر.ع"
    يتحدث الرصيد فوراً
```

## التدفق ٣ — المستثمر (المضيف)

المستثمر هو عميل قُبل طلبه من إدارة واط. يحتفظ بجميع ميزات العميل ويكتسب ثلاثة أشياء إضافية: تبويباً مخصصاً "شاحني"، وتبويب "الأرباح"، وجهاز Tuya الذكي الذي تقدمه واط.

### ٣.١ قبول الطلب

```
العميل يرسل نموذج طلب المستثمر
    │
    │ تُحفظ البيانات في جدول charger_applications
    │ الحالة: pending (معلق)
    ▼
الإدارة تراجع الطلب (راجع التدفق ٤.١)
    │
    │ الإدارة تضغط "قبول"
    ▼
النظام تلقائياً:
    • يُعيّن profile.role = 'investor'
    • يُنشئ صف charger_listings لهذا المستثمر
    • حالة الطلب ← approved (مقبول)
    │
    ▼
المستثمر يفتح التطبيق في المرة القادمة
    │
    │ التنقل يتحول إلى InvestorNavigator
    ▼
نافذة الترحيب بالمستثمر (مرة واحدة فقط)
┌──────────────────────────────────────┐
│  ✅ تمت الموافقة على طلبك! 🎉        │
│  أنت الآن مستثمر في واط             │
│                                      │
│  ⭐ موقع شاحنك:                     │
│     السيب، مسقط                     │
│                                      │
│  أكمل الإعداد لبدء قبول الحجوزات   │
│  والكسب.                             │
│                                      │
│  [متابعة]                            │
└──────────────────────────────────────┘
    │ اضغط "متابعة"
    ▼
investor_welcomed = true (لا تظهر النافذة مجدداً)
    │
    ▼
الرئيسية للمستثمر — ٥ تبويبات:
    [الخريطة] [الحجوزات] [شاحني] [الأرباح] [الملف]
```

### ٣.٢ تسليم جهاز Tuya وإعداده

هذه هي خطوة الإعداد الأكثر أهمية. تتولى إدارة واط **كل شيء** — التسليم الفيزيائي، والتركيب، والاقتران بـ Wi-Fi، واسترجاع معرّف الجهاز. الإجراء الوحيد المطلوب من المستثمر هو **لصق رمز واحد** في تطبيق واط.

#### لماذا جهاز Tuya؟

جهاز Tuya عبارة عن مفتاح ذكي متصل بـ Wi-Fi أو وحدة تحكم في شاحن EV. يتيح لواط (وللمستثمر) تشغيل الشاحن أو إيقافه عن بُعد. بدونه، لا يستطيع العملاء بدء جلسة شحن عبر التطبيق.

#### توزيع المهام دفعةً واحدة

| المسؤول | ما يفعله |
|---------|---------|
| **الإدارة** | تُسلّم الجهاز · تُركّبه · تُقرنه بـ Wi-Fi · تسترجع معرّف الجهاز · تُرسله للمستثمر |
| **المستثمر** | يستلم المعرّف · يفتح تطبيق واط · يلصق الرمز · يحفظ |

## من البداية للنهاية: دورة حياة جهاز Tuya الكاملة

يوضح هذا المخطط الرحلة الكاملة لجهاز Tuya من موافقة الإدارة إلى بدء العميل لجلسة الشحن.

```
═══════════════════════════════════════════════════════════════════
                    دورة حياة جهاز Tuya الكاملة
═══════════════════════════════════════════════════════════════════

المرحلة ١ — الطلب والقبول
────────────────────────────────────────────
العميل    ──► يُرسل نموذج طلب المستثمر
                    │
                    ▼
الإدارة   ──► تراجع في AdminInvestorsScreen
                    │
                    │ تضغط "قبول"
                    ▼
النظام    ──► profile.role = 'investor'
          ──► إنشاء صف charger_listings
          ──► حالة الطلب = 'approved'

المرحلة ٢ — الإدارة تزور الموقع وتتولى الإعداد الكامل  (خارج التطبيق)
──────────────────────────────────────────────────────────────────────
الإدارة   ──► تزور موقع المستثمر بحقيبة مستثمر واط
          ──► تُركّب المفتاح الذكي على دائرة الشاحن
          ──► تُقرن الجهاز بـ Wi-Fi المستثمر عبر Smart Life
          ──► تسترجع معرّف الجهاز من Smart Life
          ──► تُرسل المعرّف للمستثمر عبر واتساب / SMS
          ──► تختبر التشغيل/الإيقاف على الموقع ✅

المرحلة ٣ — المستثمر يلصق الرمز  (تطبيق واط — إجراء واحد فقط)
────────────────────────────────────────────────────────────────
المستثمر  ──► يستلم المعرّف من الإدارة (مثال: bf3a8c0e12345678)
          ──► يفتح واط ← تبويب "شاحني"
          ──► يضغط "إضافة معرّف الجهاز"
          ──► يلصق: bf3a8c0e12345678
          ──► يحفظ ← يُخزَّن tuya_device_id في قاعدة البيانات

              حالة الجهاز: ⏳ بانتظار التحقق

المرحلة ٤ — الإدارة تتحقق عن بُعد  (لوحة الإدارة)
─────────────────────────────────────────────────────
الإدارة   ──► تفتح AdminInvestorsScreen
          ──► تجد المستثمر ← ترى المعرّف الملصوق
          ──► تضغط "التحقق من الجهاز" (التركيب مؤكد من الزيارة)
          ──► tuya_verified = true ✅
          ──► تُبلّغ المستثمر: "تم التحقق — يمكنك الانطلاق!"

المرحلة ٦ — الانطلاق
─────────────────────
المستثمر  ──► حالة الجهاز: "تم التحقق ✓"
          ──► يُشغّل مفتاح الاتصال
          ──► is_available = true
          ──► دبوس الشاحن يظهر على الخريطة لجميع المستخدمين

المرحلة ٧ — العميل يحجز ويشحن
──────────────────────────────────
العميل    ──► يرى الشاحن على الخريطة
          ──► يحجز خانة زمنية
          ──► يصل ← "بدء الشحن الآن"
                    │
                    ▼
النظام    ──► يستدعي Tuya API: switch_1 = true (الشاحن مُشغَّل)
          ──► يُنشئ صف charging_sessions (الحالة: active)
          ──► ChargingScreen يعرض kWh / التكلفة لحظياً

العميل    ──► تنتهي الجلسة (إيقاف يدوي أو انتهاء المدة)
                    │
                    ▼
النظام    ──► يستدعي Tuya API: switch_1 = false (الشاحن متوقف)
          ──► الجلسة مكتملة، يُخصم التكلفة من المحفظة
          ──► 90% أرباح ← محفظة المستثمر
          ──► 10% عمولة ← واط

═══════════════════════════════════════════════════════════════════
```

## جداول مرجعية للحالات

### حالة الحجز

| الحالة | المعنى | العميل يرى |
|--------|--------|------------|
| `pending` | مُنشأ لكن لم يُؤكَّد | جار التحميل... |
| `confirmed` | مؤكد، ينتظر الوصول | رمز QR + عداد تنازلي |
| `active` | جلسة الشحن جارية | شاشة الشحن |
| `completed` | انتهت الجلسة | الإيصال في السجل |
| `cancelled` | ألغاه العميل | شارة ملغى |
| `no_show` | لم يصل العميل | شارة لم يحضر |

### حالة طلب المستثمر

| الحالة | المعنى | المستثمر يرى في ملفه الشخصي |
|--------|--------|------------------------------|
| `pending` | مُقدَّم، ينتظر الإدارة | "قيد المراجعة" |
| `under_review` | الإدارة تراجعه فعلياً | "قيد المراجعة" |
| `approved` | مقبول — تم منح دور المستثمر | نافذة الترحيب (مرة واحدة) |
| `rejected` | غير مقبول | "غير مقبول" |
| `needs_info` | الإدارة تحتاج وثائق إضافية | تعليق الإدارة يظهر |

### حالة جهاز Tuya

| الشرط | تسمية الحالة | يمكن الاتصال؟ |
|-------|-------------|---------------|
| لم يُدخل `tuya_device_id` | "لم يُربط بعد" | ❌ لا |
| `tuya_device_id` موجود، `tuya_verified = false` | "بانتظار تحقق الإدارة" | ❌ لا |
| `tuya_verified = true` | "تم التحقق ✓" | ✅ نعم |

### حالة المحطة

| الحالة | اللون على الخريطة | المعنى |
|--------|------------------|--------|
| `available` | 🟢 أخضر | فيها موصلات متاحة |
| `busy` | 🟠 برتقالي | جميع الموصلات مشغولة |
| `fault` | 🔴 أحمر | تم الإبلاغ عن عطل |
| `offline` | ⚫ رمادي | المحطة غير تشغيلية |
