## ١. نظرة عامة على المشروع

يحل تطبيق واط مشكلتين في آنٍ واحد:

- **أصحاب السيارات الكهربائية** في عُمان لا يملكون طريقة موثوقة للعثور على شاحن وحجزه والدفع مقابله من تطبيق واحد.
- **أصحاب العقارات** الذين لديهم شواحن منزلية معطّلة لا يملكون وسيلة لمشاركتها بأمان وكسب دخل منها.

يربط التطبيق الطرفين. يفتح العميل الخريطة، يجد أقرب محطة أو شاحن خاص، يؤكد موعداً، يصل، يمسح رمز QR، يشحن، ويدفع — كل ذلك دون نقود أو تنسيق يدوي. يتقدم المستثمر مرةً واحدة، تتم الموافقة عليه من إدارة واط، يدرج شاحنه، ويحصل على أرباحه في محفظته الرقمية.

### المنصات المدعومة

| المنصة | الحالة |
|--------|--------|
| iOS    | مدعومة |
| Android | مدعومة |
| الويب  | محدودة (صفحة تحديد الموقع فقط) |

### اللغات

| اللغة | الاتجاه |
|-------|---------|
| الإنجليزية | من اليسار إلى اليمين |
| العربية | من اليمين إلى اليسار (مع عكس كامل للواجهة) |

### تدفق البيانات عند الحجز

```
المستخدم يختار محطة → شاشة الحجز
  → التحقق من رصيد المحفظة ≥ التكلفة المتوقعة
  → إدراج صف في جدول bookings (الحالة: pending → confirmed)
  → الانتقال إلى ActiveBookingScreen (يعرض QR + العداد التنازلي)
  → يصل المستخدم، يضغط "بدء الشحن الآن"
    → (Tuya) تشغيل الشاحن
    → إدراج صف في charging_sessions (الحالة: active)
    → الانتقال إلى ChargingScreen (عرض kWh / التكلفة لحظياً)
  → المستخدم يوقف أو تنتهي المدة
    → تحديث charging_sessions (completed, kwh_delivered, cost)
    → خصم التكلفة من profiles.wallet_balance
    → إدراج صف في wallet_transactions (النوع: charge)
    → الانتقال إلى SessionSummaryScreen (إيصال + CO₂ موفّر)
```

## ٥. البيئة والإعدادات

بيانات اعتماد Supabase مدرجة حالياً مباشرةً في `src/lib/supabase.ts`. في بيئة الإنتاج، انقلها إلى متغيرات البيئة عبر `app.config.js` بصيغة `EXPO_PUBLIC_`.

### الإعداد الحالي (`src/lib/supabase.ts`)

```typescript
const SUPABASE_URL      = 'https://cnwlmbpmgwmhzzjnmltz.supabase.co';
const SUPABASE_ANON_KEY = '<anon-key>';

export const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY, {
  auth: {
    storage:            AsyncStorage,  // استمرار الجلسة بعد إعادة تشغيل التطبيق
    autoRefreshToken:   true,
    persistSession:     true,
    detectSessionInUrl: false,         // معطّل — الروابط العميقة تُعالج يدوياً
  },
});
```

### نظام الروابط العميقة (Deep Links)

يستخدم التطبيق النظام `watt://` لاستقبال ردود OAuth:

| الحدث | الرابط العميق |
|-------|--------------|
| ردّ Google OAuth | `watt://auth/callback` |
| ردّ إعادة تعيين كلمة المرور | `watt://reset-password` |

يُضبط هذا النظام في `app.json` تحت `expo.scheme`.

## ٦. تدفقات المصادقة

تُدار جميع عمليات المصادقة بواسطة `src/context/AuthContext.tsx`. يوفر السياق واجهة برمجية موحدة لكل شاشة عبر خطاف `useAuth()`.

### ٦.١ البريد الإلكتروني وكلمة المرور

**إنشاء الحساب**

١. يدخل المستخدم الاسم الكامل والبريد الإلكتروني وكلمة المرور.
٢. `supabase.auth.signUp()` تنشئ مستخدماً جديداً.
٣. يُدرج صف في `profiles` بـ `full_name` و`wallet_balance: 0` و`is_active: true`.
٤. `fetchProfile()` يحمّل الملف الشخصي؛ ينتقل التنقل إلى CustomerNavigator.

**تسجيل الدخول**

١. يدخل المستخدم البريد الإلكتروني وكلمة المرور.
٢. `supabase.auth.signInWithPassword()` تعيد جلسة.
٣. `fetchProfile()` يحمّل الملف؛ إن كان `is_active === false` يُحظر الحساب ويُسجَّل الخروج فوراً.

### ٦.٢ تسجيل الدخول بـ Google

١. `supabase.auth.signInWithOAuth({ provider: 'google', ... })` تعيد رابط التفويض.
٢. `WebBrowser.openAuthSessionAsync()` يفتح متصفح داخل التطبيق.
٣. عند النجاح، يُغلق المتصفح ويُعيد `result.url` مع رمز التفويض.
٤. `supabase.auth.exchangeCodeForSession(result.url)` يستبدل الرمز بجلسة.
٥. `ensureProfile()` تُنشئ صف الملف الشخصي إن لم يكن موجوداً (لأول مرة عبر OAuth).

### ٦.٣ تسجيل Apple (iOS فقط)

١. `AppleAuthentication.signInAsync()` تعيد بيانات اعتماد تحتوي على `identityToken`.
٢. `supabase.auth.signInWithIdToken({ provider: 'apple', token })` يتحقق من الرمز وينشئ جلسة.
٣. `ensureProfile()` تُنشئ الملف الشخصي.

> تسجيل Apple محمي بـ `Platform.OS !== 'ios'` و`isAvailableAsync()`. على Android يظهر "غير متاح على هذا الجهاز".

### ٦.٤ نسيت كلمة المرور

١. يدخل المستخدم بريده الإلكتروني.
٢. `supabase.rpc('check_email_exists', { p_email })` يتحقق أن الحساب موجود قبل الإرسال. إن لم يُوجد يظهر "لا يوجد حساب بهذا البريد".
٣. `supabase.auth.resetPasswordForEmail()` يُرسل رابط إعادة التعيين.

### ٦.٥ تجاوز تسجيل الدخول (وضع المطور)

يكشف `AuthContext` عن `devSignIn(profile)` / `devSignOut()`. تتيح `DevLoginScreen` للمطورين محاكاة أي دور (عميل، مستثمر، إدارة) دون إنشاء حسابات حقيقية.

## ٧. أدوار المستخدمين والتنقل

يتبدّل المتصفح الجذري (`src/navigation/index.tsx`) بين أربعة متصفحات فرعية بناءً على `profile.role`.

```
profile.role          → المتصفح
──────────────────────────────────────────────────
(غير مسجّل)           → GuestNavigator
'customer'            → CustomerNavigator
'investor' | 'host'  → InvestorNavigator
'admin'               → AdminNavigator
```

### ٧.١ متصفح الضيف (GuestNavigator)

يصل إليه غير المسجّلين. يمكنهم تصفح الخريطة وعرض تفاصيل المحطات. تبويبا الحجوزات والمحفظة يظهران شاشة مقفلة مع دعوة لتسجيل الدخول.

```
GuestStack
  ├── SignIn (تسجيل الدخول)
  ├── SignUp (إنشاء حساب)
  └── GuestTabs (التبويبات السفلية)
        ├── GuestMap       ← MapScreen
        ├── GuestBookings  ← GuestLockedScreen
        ├── GuestWallet    ← GuestLockedScreen
        └── GuestProfile   ← GuestProfileScreen
```

### ٧.٢ متصفح العميل (CustomerNavigator)

```
CustomerStack
  ├── Tabs (التبويبات السفلية)
  │     ├── Map            ← MapScreen
  │     ├── Bookings       ← BookingsScreen
  │     ├── Wallet         ← WalletScreen
  │     └── Profile        ← ProfileScreen
  ├── StationDetails       ← StationDetailsScreen
  ├── Booking              ← BookingScreen
  ├── ActiveBooking        ← ActiveBookingScreen
  ├── Charging             ← ChargingScreen
  ├── SessionSummary       ← SessionSummaryScreen (الإيماءة معطّلة)
  └── InvestorApplication  ← InvestorApplicationScreen
```

### ٧.٣ متصفح المستثمر (InvestorNavigator)

للمستثمرين جميع شاشات العملاء بالإضافة إلى تبويبين مخصصين.

```
InvestorStack
  ├── InvestorTabs (التبويبات السفلية)
  │     ├── Map              ← MapScreen
  │     ├── Bookings         ← BookingsScreen
  │     ├── InvestorCharger  ← InvestorChargerScreen (شاحني)
  │     ├── Wallet           ← InvestorEarningsScreen (الأرباح)
  │     └── Profile          ← ProfileScreen
  └── (نفس شاشات سكة CustomerStack)
```

تظهر **نافذة الترحيب بالمستثمر** مرةً واحدة بعد موافقة الإدارة (`investor_welcomed === false`). تعرض عنوان الشاحن وزراً "متابعة". عند الإغلاق يُعيَّن `investor_welcomed = true` في الملف الشخصي.

### ٧.٤ متصفح الإدارة (AdminNavigator)

```
AdminStack
  └── AdminTabs (لون التمييز: بنفسجي #7C3AED)
        ├── AdminMap       ← AdminMapScreen
        ├── AdminCustomers ← AdminUsersScreen
        ├── AdminInvestors ← AdminInvestorsScreen
        └── AdminProfile   ← AdminProfileScreen
```

## ٨. الشاشات والميزات

### ٨.١ شاشة البداية (SplashScreen)

ثلاثة شرائح تعريفية تظهر عند أول تشغيل:

| الشريحة | العنوان | الوصف |
|---------|---------|-------|
| ١ | شحن سريع وذكي | ابحث عن أقرب محطة وابدأ جلستك في ثوانٍ |
| ٢ | محطات في كل مكان | شبكة متنامية عبر السلطنة |
| ٣ | ادفع بسهولة | محفظة رقمية متكاملة |

أزرار التنقل: تخطي · التالي · ابدأ الآن.

### ٨.٢ شاشة اختيار الدور (RoleSelectScreen)

خياران:
- **أريد الشحن** → مسار التسجيل / الدخول للعميل
- **لديّ شاحن** → مسار طلب المستثمر

### ٨.٣ شاشة تسجيل الدخول (SignInScreen)

الحقول: البريد الإلكتروني، كلمة المرور.  
الإجراءات: تسجيل الدخول · نسيت كلمة المرور · Google · Apple (iOS) · التصفح كضيف · الانتقال إلى إنشاء الحساب.

### ٨.٤ شاشة إنشاء الحساب (SignUpScreen)

الحقول: الاسم الكامل، البريد الإلكتروني، كلمة المرور.  
الإجراءات: إنشاء الحساب · Google · Apple (iOS) · الانتقال إلى تسجيل الدخول.

الكشف عن البريد المكرر: يتحقق من `data.user.identities.length === 0` ورسالة خطأ Supabase.

### ٨.٥ شاشة الخريطة (MapScreen)

التبويب الرئيسي لجميع الأدوار. يعرض:
- جميع المحطات العامة من جدول `stations` (مرمّزة بالألوان حسب الحالة: أخضر=متاح، برتقالي=مشغول، أحمر=عطل، رمادي=غير متصل).
- جميع قوائم الشواحن الخاصة النشطة من `charger_listings` (مع تمييز "شاحني" للمستثمر على خاصته).
- قائمة سفلية للمحطات القريبة مرتبة بالمسافة.
- شريط بحث لتصفية المحطات بالاسم.

النقر على دبوس الخريطة أو عنصر القائمة يُفضي إلى `StationDetails`.

### ٨.٦ شاشة تفاصيل المحطة (StationDetailsScreen)

تعرض معلومات المحطة الكاملة:
- الاسم، العنوان، شارة الحالة
- السعر لكل kWh، قدرة الطاقة، التقييم
- أنواع الموصلات مع حالة كل منها
- ساعات التشغيل، آخر صيانة
- المرافق (أيقونات: مول، واي فاي، مواقف، أكل، إلخ)
- زر "احجز الآن" ← `BookingScreen`
- زر "الاتجاهات" ← يفتح خرائط Google / Apple Maps

المحطات بحالة غير متاح تظهر زراً معطلاً "غير متاح".

### ٨.٧ شاشة الحجز (BookingScreen)

إنشاء الحجز خطوةً خطوة:

١. **اختيار اليوم** — شريط أفقي للأيام السبعة القادمة.
٢. **اختيار الوقت** — شبكة من خانات زمنية كل ٣٠ دقيقة.
٣. **شريط المدة** — من ٣٠ دقيقة إلى ٢٤٠ دقيقة بخطوات ٣٠.
٤. **بطاقة ملخص التكلفة** — kWh المتوقعة، السعر، التكلفة الإجمالية، الرصيد بعد الحجز.

حارس المحفظة: إن كان `wallet_balance < estimated_cost` تظهر نافذة لشحن المحفظة.

عند التأكيد: إدراج صف في `bookings` والانتقال إلى `ActiveBookingScreen`.

### ٨.٨ شاشة الحجز النشط (ActiveBookingScreen)

تعرض حالة الحجز المؤكد:
- رمز QR كبير (مولّد من UUID في `bookings.qr_code`).
- عداد تنازلي للموعد المحجوز.
- تفاصيل الحجز (المحطة، التاريخ، الوقت، المدة، التكلفة).
- زر "بدء الشحن الآن" — يُشغّل الشاحن (مفتاح Tuya) ويبدأ صف `charging_sessions`.
- زر "إلغاء الحجز" — يفتح نافذة اختيار السبب ثم يُحدّث `bookings.status = 'cancelled'`.

### ٨.٩ شاشة الشحن (ChargingScreen)

عرض جلسة الشحن الحية، مشتركة في التحديثات الفورية:
- أيقونة بطارية متحركة.
- kWh المُسلَّمة، التكلفة الجارية، المدة المنقضية.
- الرصيد المتبقي في المحفظة بعد الجلسة.
- زر "إيقاف الشحن" — ينهي الجلسة ويحسب التكلفة النهائية.
- "العودة للرئيسية" (تصغير) — تستمر الجلسة في الخلفية مع ظهور لافتة عائمة.

### ٨.١٠ شاشة ملخص الجلسة (SessionSummaryScreen)

إيصال ما بعد الجلسة:
- kWh المُسلَّمة، التكلفة الإجمالية، المدة.
- حساب CO₂ الموفَّر: `kWh × 0.233 كجم` (مقارنةً بسيارة بنزين).
- إجراء "مشاركة الملخص".
- "العودة للرئيسية" تُعيد التنقل إلى تبويب الخريطة.

إيماءة السحب للخلف معطّلة (`gestureEnabled: false`) لمنع التنقل غير المقصود من الإيصال.

### ٨.١١ شاشة الحجوزات (BookingsScreen)

تسرد جميع حجوزات المستخدم المسجّل. الأقسام:
- **القادمة** (مؤكدة / نشطة، مرتبة تصاعدياً).
- **السابقة** (مكتملة / ملغاة، مرتبة تنازلياً).

شارات التصفية: الكل · النشطة · المؤكدة · المكتملة.

تُلغى الحجوزات عبر نافذة اختيار السبب ← `bookings.status = 'cancelled'`.

### ٨.١٢ شاشة المحفظة (WalletScreen)

تعرض:
- الرصيد الحالي (ر.ع).
- إحصاءات: إجمالي الجلسات، إجمالي kWh، إجمالي المُنفق.
- زر "شحن" ← نافذة بمبالغ محددة مسبقاً.
- سجل المعاملات مع شارات تصفية (الكل · شحن · دفع · استرداد).

### ٨.١٣ شاشة الملف الشخصي (ProfileScreen)

مركز ملف المستخدم:
- الصورة الشخصية (النقر عليها يُغيّرها).
- الاسم الكامل، البريد، الهاتف، طراز السيارة.
- شارة العضوية (عادية / فضية / ذهبية) مع شريط تقدم الجلسات.
- قسم المركبة: الطراز + نوع الموصل + السنة.
- معاينة سجل الشحن.
- **بطاقة دعوة المستثمر** — تعرض حالة الطلب الحالي أو دعوة "كن مستثمراً في واط".
- الإعدادات: الإشعارات، الأمن والخصوصية، المساعدة، حول التطبيق، تبديل اللغة.
- تسجيل الخروج / حذف الحساب.

### ٨.١٤ شاشة طلب الاستثمار (InvestorApplicationScreen)

نموذج متعدد الأقسام يستخدمه العملاء للتقدم كمستثمرين:

| القسم | الحقول |
|-------|--------|
| المعلومات الشخصية | الاسم الكامل*، رقم الهاتف* |
| الموقع | أداة تحديد الموقع على الخريطة (مع عكس الترميز الجغرافي تلقائياً) |
| اسم المحطة | اختياري |
| تفاصيل الشاحن | النوع*، قدرة الطاقة (kW) |
| المتطلبات الحكومية | اسم نموذج الكهرباء*، رقم السجل التجاري*، رقم بطاقة الهوية* |

عند الإرسال: إدراج صف في `charger_applications` (الحالة: `pending`).  
حالة النجاح تعرض قائمة تحقق من ٤ خطوات: مراجعة الطلب ← زيارة الموقع ← توقيع العقد ← تركيب الشاحن.

### ٨.١٥ شاشة الضيف المقفل (GuestLockedScreen)

تظهر بدلاً من الحجوزات أو المحفظة لغير المسجّلين. تسرد المزايا المتاحة بعد التسجيل وتوفر زرَّي "تسجيل الدخول" و"إنشاء حساب".

## ٩. تدفق الحجز

```
شاشة الخريطة
  └─► تفاصيل المحطة
          └─► شاشة الحجز
                  │
                  ├── [رصيد غير كافٍ] ← نافذة شحن المحفظة
                  │
                  └── [تأكيد] ← شاشة الحجز النشط
                                    │
                                    ├── [إلغاء] ← شاشة الحجوزات
                                    │
                                    └── [بدء الشحن]
                                            │
                                            ▼
                                      شاشة الشحن
                                            │
                                      [إيقاف / انتهاء تلقائي]
                                            │
                                            ▼
                                    شاشة ملخص الجلسة
                                            │
                                      [العودة للرئيسية]
                                            │
                                            ▼
                                      شاشة الخريطة
```

### دورة حياة حالة الحجز

```
pending (معلق) → confirmed (مؤكد) → active (نشط) → completed (مكتمل)
                                  ↘ cancelled (ملغى)
                                  ↘ no_show (لم يحضر)
```

## ١٠. تدفق المستثمر (المضيف)

### ١٠.١ تقديم الطلب

١. يفتح العميل `ProfileScreen` ويضغط "كن مستثمراً في واط".
٢. تُفتح `InvestorApplicationScreen`. يملأ العميل البيانات الشخصية، يحدد الموقع على الخريطة، يحدد نوع الشاحن والوثائق الحكومية.
٣. يُرسَل النموذج إلى جدول `charger_applications` (الحالة: `pending`).
٤. تتحول بطاقة الملف الشخصي إلى "الطلب قيد المراجعة".

### ١٠.٢ مراجعة الإدارة

١. تفتح الإدارة `AdminInvestorsScreen` وترى الطلب الجديد.
٢. يمكن للإدارة: **قبول** · **رفض** · **تعليق قيد المراجعة** · **إضافة تعليق للمتقدم** · **حذف الطلب**.
٣. عند **القبول**: تُنفَّذ RPC (`accept_investor_application`) تُعيّن `profiles.role = 'investor'` وتُنشئ صفاً في `charger_listings`.

### ١٠.٣ ما بعد الموافقة

١. يفتح المستثمر التطبيق ← ينتقل التنقل إلى `InvestorNavigator`.
٢. تظهر **نافذة الترحيب بالمستثمر** مرةً واحدة (تعرض عنوان الشاحن ثم تُعيّن `investor_welcomed = true`).
٣. يظهر للمستثمر التبويب الجديد "شاحني".

### ١٠.٤ شاشة شاحن المستثمر (InvestorChargerScreen)

| الميزة | الوصف |
|--------|-------|
| مفتاح متصل / غير متصل | يُعيّن `charger_listings.is_available`. يظهر الدبوس على الخريطة للعملاء فقط عند الاتصال. |
| معرّف جهاز Tuya | المستثمر يدخل معرّف جهازه ويحفظه في `tuya_device_id`. |
| التحقق من الإدارة | الإدارة تُعيّن `tuya_verified = true` بعد التحقق. |
| تعديل التفاصيل | نافذة لتحديث العنوان، الطاقة (kW)، السعر، ساعات التوفر، الملاحظات. |
| حجوزات العملاء | قائمة حية بجميع الحجوزات على قائمة المستثمر. |
| شحن سيارتي | المستثمر يمكنه بدء جلسة شحن لنفسه باستخدام شاحنه. |

### ١٠.٥ شاشة أرباح المستثمر (InvestorEarningsScreen)

- الرصيد المتاح (من `profiles.wallet_balance`).
- الأرباح هذا الشهر مقابل الإجمالي الكلي.
- زر "السحب" (قريباً).
- سجل المعاملات المصفّى لحساب المستثمر.

## ١٥. دعم اللغات (i18n)

### ملفات اللغات

| الملف | اللغة |
|-------|-------|
| `src/i18n/en.ts` | الإنجليزية (LTR) |
| `src/i18n/ar.ts` | العربية (RTL) |

كلا الملفين يصدّران نفس واجهة `Translations` المُعرَّفة في `ar.ts`. كل نص في التطبيق — بما فيه تسميات الحالات، رسائل الأخطاء، نصوص الأزرار، وإجابات الأسئلة الشائعة — له مفتاح ترجمة.

### سياق اللغة (LanguageContext)

```typescript
const { t, isRTL, lang, setLang } = useLang();
```

- `t` — كائن الترجمات الكامل للغة النشطة.
- `isRTL` — `true` عند تفعيل العربية؛ يُستخدم لعكس `flexDirection` و`textAlign` وأيقونات الاتجاه في جميع الشاشات.
- `lang` — `'en'` | `'ar'`.
- `setLang(lang)` — يحفظ الاختيار في `AsyncStorage`.

تختزن تفضيلات اللغة عبر إعادة تشغيل التطبيق.

### خريطة المحافظات (`src/i18n/govMap.ts`)

تعيين محافظات عُمان الإحدى عشرة بين العربية والإنجليزية لاستخدامها في نموذج طلب المستثمر:

```typescript
'مسقط' → 'Muscat'
'ظفار' → 'Dhofar'
// وهكذا
```

- توسيع الصف لعرض معلومات المركبة.
- الإجراءات: **تعطيل** (يعيّن `is_active = false`) · **إعادة التفعيل** · **حذف** (حذف نهائي مع تأكيد).

### AdminInvestorsScreen — إدارة المستثمرين

- قائمة قابلة للبحث لجميع `charger_applications` مع شارات تصفية: الكل · معلق · مقبول · مرفوض.

## ١٨. تكامل الشاحن الذكي (Tuya)

تستخدم واط سحابة Tuya IoT للتحكم عن بُعد في شواحن المستثمرين.

### تدفق الإعداد

١. يشتري المستثمر شاحن EV أو مفتاح ذكي متوافق مع Tuya.
٢. يربط المستثمر الجهاز بحسابه في Tuya عبر تطبيق Tuya / Smart Life.
٣. في `InvestorChargerScreen`، يدخل المستثمر **معرّف جهاز Tuya** (سلسلة سداسية عشرية من 16 حرفاً).
٤. يُحفظ المعرّف في `charger_listings.tuya_device_id`.
٥. تراجع الإدارة وتعيّن `tuya_verified = true`.
٦. أصبح بالإمكان التحكم في الشاحن عن بُعد.

### التبديل عن بُعد

عند ضغط العميل "بدء الشحن الآن":
١. يستدعي التطبيق Edge Function (أو RPC) التي تستدعي Tuya API: `PUT /v1.0/devices/{device_id}/commands` مع `{ code: 'switch_1', value: true }`.
٢. يُشغَّل الشاحن.
٣. يُنشأ صف `charging_sessions` (الحالة: `active`).

عند انتهاء الجلسة:
١. استدعاء Tuya API يعيّن `switch_1: false` (إيقاف الشاحن).
٢. يُحدَّث صف الجلسة (الحالة: `completed`، `kwh_delivered`، `cost`).

### مؤشرات الحالة في InvestorChargerScreen

| الحالة | التسمية | المعنى |
|--------|---------|--------|
| لا يوجد `tuya_device_id` | "لم يُربط بعد" | لم يدخل المستثمر معرّف الجهاز |
| تم تعيين `tuya_device_id`، `tuya_verified = false` | "بانتظار تحقق الإدارة" | ينتظر الإدارة |
| `tuya_verified = true` | "تم التحقق ✓" | الجهاز جاهز |

## ٢٠. المحفظة والمدفوعات

### شحن المحفظة

يضيف العملاء رصيداً عبر **ثواني** (بوابة الدفع العُمانية المحلية).

المبالغ المحددة مسبقاً: 1 · 2 · 5 · 10 · 20 · 50 ر.ع.

التدفق:
١. يضغط العميل "شحن" في شاشة المحفظة.
٢. يختار المبلغ.
٣. النافذة تعرض: الرصيد الحالي / المبلغ المضاف / الرصيد الجديد / الدفع عبر ثواني.
٤. عند التأكيد: يفتح نموذج دفع ثواني.
٥. عند النجاح: يزيد `wallet_balance`، يُدرج صف `wallet_transactions` (النوع: `topup`).

### خصم الشحن

عند انتهاء الجلسة، يُخصم `cost` من `wallet_balance` ويُدرج صف `wallet_transactions` (النوع: `charge`).

### التسعير

| نوع المحطة | السعر لكل kWh |
|-----------|--------------|
| شاحن سريع (150 kW) | 0.028–0.030 ر.ع |
| شاحن عادي (22–50 kW) | 0.024–0.028 ر.ع |
| شاحن خاص (مستثمر) | يحدده المستثمر (الافتراضي 0.025 ر.ع) |

**عمولة واط**: 10% من جميع الإيرادات من جلسات شواحن المستثمرين.
