## 1. العقبات الحرجة

هذه العقبات ستُعطّل التطبيق أو تمنع المستخدمين من إتمام الإجراءات الأساسية. يجب إصلاحها قبل أي إطلاق.

### B-01 — الدفع عبر ثواني غير مُفعّل

**الخطورة:** 🔴 حرجة
**ما يفعله الكود فعلياً:** ملف `WalletScreen.tsx` السطر 87 يكتب `payment_method: 'thawani'` مباشرةً في قاعدة البيانات ويُحدّث رصيد المحفظة — **دون الاستدعاء الفعلي لـ API ثواني**. لا يوجد SDK لثواني، ولا جلسة دفع، ولا إعادة توجيه. يمكن للمستخدمين الشحن مجاناً الآن.
**ما يجب بناؤه:**
- تسجيل حساب تاجر في Thawani Pay (thawani.om)
- دمج Thawani Checkout API: إنشاء جلسة دفع ← إعادة توجيه المستخدم لصفحة الدفع ← استقبال Webhook ← تحديث المحفظة عند تأكيد الدفع فقط
- إنشاء Edge Function في Supabase لاستقبال الـ Webhook (لتجنّب كشف المفتاح السري في التطبيق)

| البند | التكلفة |
|-------|---------|
| حساب التاجر في ثواني | مجاني |
| رسوم معاملات ثواني | 2.9% من كل عملية شحن + 0.100 ريال عُماني رسوم ثابتة |
| عمل تطوير | 2–4 أيام |

> **مثال:** إذا شحن مستخدم 10 ريال عُماني ← تأخذ ثواني ~0.385 ريال ← يتلقى المستثمر/Watt 9.615 ريال.

### B-02 — الإشعارات الفورية غير مُنفّذة

**الخطورة:** 🔴 حرجة
**ما يفعله الكود فعلياً:** `ProfileScreen.tsx` يحتوي على مفاتيح تبديل لـ "إشعارات فورية" و"تذكيرات الحجوزات" و"تحديثات الشحن". هذه مجرد متغيرات حالة في الواجهة — لا رموز إشعارات مسجّلة، ولا رسائل مُرسَلة.
تقوم Edge Function الموجودة في `supabase/functions/notify-booking/index.ts` لكن لم تُضبط بها بيانات اعتماد FCM/APNs.
**ما يجب بناؤه:**
- إضافة حزمة `expo-notifications`
- طلب إذن الإشعارات عند أول تشغيل
- تخزين رمز Expo Push Token في جدول `profiles`
- ربط Edge Function ببيانات اعتماد Expo Push أو مشروع Firebase
- إرسال إشعارات تأكيد الحجز، وبدء جلسة الشحن، وتنبيه المستثمر عند حجز جديد

| البند | التكلفة |

| حزمة `expo-notifications` | مجاني |
| مشروع Firebase (FCM) | مجاني حتى مليون رسالة/شهر |

### B-04 — تكامل Tuya يستخدم حزمة Node.js لا تعمل في React Native

**الخطورة:** 🔴 حرجة
**ما يفعله الكود فعلياً:** `package.json` يُدرج `@tuya/tuya-connector-nodejs` — وهذا **SDK خادم Node.js**. لا يمكن تشغيله داخل تطبيق React Native. كما أن Supabase Edge Functions تعمل على **Deno** وليس Node.js، فالحزمة لن تعمل هناك أيضاً.
**ما يجب بناؤه:**
- إنشاء Edge Function في Supabase تستدعي **Tuya Open API** مباشرةً عبر HTTP fetch (بدون SDK)
- تحتاج Edge Function إلى: متغيري بيئة `TUYA_ACCESS_ID` و`TUYA_ACCESS_SECRET`
- التطبيق يستدعي Edge Function؛ وهي بدورها تستدعي API تويا لتشغيل/إيقاف الجهاز
- توقيع طلبات Tuya API بـ HMAC-SHA256 (مواصفات المصادقة من تويا)

| البند | التكلفة |
|-------|---------|
| حساب Tuya IoT Platform | مجاني |
| Tuya Cloud API — الطبقة المجانية | مليون استدعاء API/شهر مجاناً |
| الخطة المدفوعة (في حال التجاوز) | ~0.001$ لكل استدعاء |
| عمل تطوير | 2–3 أيام (توقيع المصادقة ليس بسيطاً) |

### B-08 — رابط إعادة تعيين كلمة المرور (`watt://reset-password`) غير مُعالَج

**الخطورة:** 🟠 مهمة
**ما يفعله الكود:** `AuthContext` يُرسل بريد إعادة تعيين مع `redirectTo: 'watt://reset-password'`. لا توجد شاشة في التنقل تستقبل هذا الرابط وتتيح للمستخدم تعيين كلمة مرور جديدة.
**ما يجب بناؤه:**
- معالجة الرابط العميق `watt://reset-password` في التطبيق (باستخدام `expo-linking`)
- إضافة `ResetPasswordScreen` يقبل الرمز من الرابط ويستدعي `supabase.auth.updateUser({ password })`
- ربطه بالمنظّم الجذري للتنقل

| البند | التكلفة |
|-------|---------|
| عمل تطوير | يوم واحد |
| التكلفة | مجاني |

### B-09 — حالة الشاحن الفوري (حالة مفتاح Tuya) لا تنعكس في الواجهة

**الخطورة:** 🟠 مهمة
**ما يفعله الكود:** حقل `switch_status` في `charger_listings` موجود لكن لا يُحدّثه أي Hook في الوقت الفعلي. إذا بدأ مستخدم آخر جلسة، لا تتحدث شاشة شاحن المستثمر تلقائياً.
**ما يجب بناؤه:**
- اشتراك Supabase Realtime على `charger_listings` لصف المستثمر
- استطلاع دوري أو Webhook من Tuya لتحديث `switch_status` بعد الأوامر عن بُعد

| البند | التكلفة |
|-------|---------|
| عمل تطوير | يوم واحد |
| التكلفة | مجاني (ضمن حدود Supabase Realtime) |

---

### B-10 — إعدادات الإشعارات لا تُحفظ ولا تُطبَّق

**الخطورة:** 🟠 مهمة
**ما يفعله الكود:** مفاتيح الإشعارات في `ProfileScreen` هي حالة React محلية (`useState`). إغلاق التطبيق يُعيدها إلى `true`. لا تُحفظ في قاعدة البيانات، ولا يُراعيها أي كود قبل إرسال الإشعارات.
**ما يجب بناؤه:**
- إضافة عمود JSON `notification_prefs` لجدول `profiles`
- حفظ حالة المفاتيح في Supabase عند تغييرها
- قراءة التفضيلات في Edge Function قبل إرسال أي إشعار

| البند | التكلفة |
|-------|---------|
| Migration: إضافة العمود | 0.5 يوم |
| التكلفة | مجاني |
